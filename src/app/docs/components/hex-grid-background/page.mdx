# Hex Grid Background

SVG-based honeycomb hexagon grid pattern. Positioned absolute to fill its parent container. Use it to keep panels, dashboards, and empty surfaces from feeling flat without introducing a heavier monitor frame.

```tsx
import { HexGridBackground } from "@mdrbx/nerv-ui";
```

<Callout type="warning" title="GENERIC TEXTURE">
  `HexGridBackground` is useful for neutral futuristic surfaces, but if you want
  a backdrop that reads more directly like NERV / MAGI monitor framing, use{" "}
  <Link href="/docs/components/monitor-overlay">Monitor Overlay</Link>.
</Callout>

<Callout type="info" title="BEST WHEN LAYERED">
  On pure black with the default low opacity, this pattern is intentionally
  subtle. It starts to read much more clearly when it sits over a lightly tinted
  surface, under labels, progress bars, or data blocks.
</Callout>

## What It Adds

The grid is not meant to be a full visual treatment by itself. Its job is to
add material texture and depth to a surface that would otherwise read as a flat
black rectangle.

## Layered Usage

This component reads best as a secondary layer underneath real interface
content. A slight tint, a brighter border, and one or two UI elements are
usually enough to make the texture feel intentional.
